# Skill Architecture: Gulin vs Victor

## Anton Gulin's Model — Code-Gen Pipeline

```
┌─────────────────────────────────────────────────┐
│                 skill-creator                    │
│  ┌──────────┐   ┌──────────┐   ┌─────────────┐ │
│  │  Create  │ → │   Eval   │ → │  Optimize   │ │
│  │ SKILL.md │   │(external)│   │             │ │
│  └──────────┘   └──────────┘   └─────────────┘ │
│                              ↓                  │
│                        ┌──────────┐             │
│                        │  Deploy  │             │
│                        │npm pkg   │             │
│                        └──────────┘             │
└─────────────────────────────────────────────────┘
                       ↓
            ┌─────────────────────┐
            │   SKILL.md          │ ← only one layer
            │ (prompts + rules)   │
            └─────────────────────┘
                       ↓
            ┌─────────────────────┐
            │   Code output       │
            │ (test, spec, impl)  │
            └─────────────────────┘

Problem: no validation layer, no stop-rules,
external judge required, no self-evaluation
```

## Victor's Model — Behavior Blueprint

```
┌────────────────────────────────────────────────────────────┐
│                    Project Context Layer                    │
│  ┌──────────────────────────────────────────────────────┐  │
│  │                AGENTS.md                              │  │
│  │  - Boundaries (edit/ask/never)                        │  │
│  │  - Sources of Truth (order)                           │  │
│  │  - Session procedures                                 │  │
│  └──────────────────────────────────────────────────────┘  │
│  ┌──────────────────────────────────────────────────────┐  │
│  │              Stop-Rules (RULES.md pattern)            │  │
│  │  - What NOT to do (anti-patterns)                    │  │
│  │  - Learned from failures (learn "...")               │  │
│  │  - Examples: context.time vs file dates, truncation   │  │
│  └──────────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────────┘
                            ↓
┌────────────────────────────────────────────────────────────┐
│                    Skill Layer                               │
│  ┌──────────────────────────────────────────────────────┐  │
│  │                SKILL.md                               │  │
│  │  - Domain knowledge, patterns                         │  │
│  │  - Example output, conventions                        │  │
│  │  - Generated by skill-creator                         │  │
│  └──────────────────────────────────────────────────────┘  │
│  ┌──────────────────────────────────────────────────────┐  │
│  │          Self-Evaluation (embedded, 4 axes)           │  │
│  │  ┌───────────┐ ┌──────────┐ ┌────────┐ ┌────────┐  │  │
│  │  │Correctness│ │Relevance │ │Stability│ │Coverage│  │  │
│  │  │ compile/  │ │ checklist│ │3-run    │ │edge    │  │  │
│  │  │ test pass │ │ match    │ │variance │ │cases   │  │  │
│  │  └───────────┘ └──────────┘ └────────┘ └────────┘  │  │
│  └──────────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────────┘
                            ↓
┌────────────────────────────────────────────────────────────┐
│                 Validation Layer (BDD Harness)              │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  Given-When-Then specs (executable)                  │  │
│  │  - Parsed by Cucumber / Behave / custom              │  │
│  │  - Run as E2E tests in CI/CD                         │  │
│  │  - Violation → build error (not stale doc)            │  │
│  │  - Agent can run 100× per session                    │  │
│  └──────────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────────┘
                            ↓
┌────────────────────────────────────────────────────────────┐
│                    Output Layer                             │
│  ┌──────────┐  ┌──────────┐  ┌─────────────────────────┐  │
│  │ Test     │  │ Feature  │  │ Agent behavior decision │  │
│  │ code     │  │ files    │  │ (ask when uncertain,    │  │
│  │          │  │          │  │  return when stuck,     │  │
│  │          │  │          │  │  fail honestly)         │  │
│  └──────────┘  └──────────┘  └─────────────────────────┘  │
└────────────────────────────────────────────────────────────┘
```

## Key Differences

```
┌────────────────────────────┬──────────────────────────────┐
│        Gulin               │          Victor              │
├────────────────────────────┼──────────────────────────────┤
│ One layer: SKILL.md        │ Three layers: Context +      │
│                            │ Skill + Validation           │
├────────────────────────────┼──────────────────────────────┤
│ External judge needed      │ Self-evaluation (4 axes)     │
├────────────────────────────┼──────────────────────────────┤
│ No stop-rules              │ Stop-rules from failures     │
├────────────────────────────┼──────────────────────────────┤
│ No validation layer        │ BDD harness = executable     │
│                            │ specs as CI gate             │
├────────────────────────────┼──────────────────────────────┤
│ Skill = code generation    │ Skill = agent behavior       │
│ package                    │ blueprint                    │
├────────────────────────────┼──────────────────────────────┤
│ Optimize → Deploy          │ Learn → Add stop-rule →      │
│                            │ Re-evaluate (RULES.md cycle) │
└────────────────────────────┴──────────────────────────────┘
```

## Flow: How Both Handle a Task

### Gulin
```
Task → SKILL.md → eval(external) → generate code → output
```

### Victor
```
Task → AGENTS.md(boundaries) + Stop-rules(check) 
     → SKILL.md(domain) + Self-eval(4 axes)
     → Generate BDD specs + code
     → Run harness → build pass/fail
     → If fail → learn → add stop-rule → retry
     → Output (test + spec + behavior guarantee)
```
