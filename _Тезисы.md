## Post 1-3: BASICS (Основы)

### Post 1: MAS-Pipeline
- 4 роли: Generator, Critic, Fixer, Executor
- Virtual QA team заменяет human QA
- Итеративный процесс до pass

#### Архитектура MAS (Апрель 2026)

```
1. Requirement Layer (DoR)     — ИИ проверяет, можно ли тестировать
2. Creative Layer (Generator)  — Создает первичный код
3. Adversarial Layer (Critic)   — Находит слабые места
4. Sandbox Layer (Docker)     — Проверяет в изоляции
5. Healing Layer (Fixer+RAG)   — Исправляет + "умная память"
6. Truth Layer (Mutation)      — Проверяет что тесты "убивают" баги
```

## Риски MAS-Pipeline (дополнено 2026-04-19)

### Риски от Ollama (Qwen 2.5) — оригинальные ответы

> "В MAS-Pipeline (Generator, Critic, Fixer, Executor) для QA - какие главные риски если Generator даёт плохие тесты?"

1. **Снижение качества тестирования** — тесты не соответствуют целям проекта
2. **Уменьшение эффективности** — Critic/Fixer тратят время на исправление вместо поиска багов
3. **Ухудшение доверия между агентами** — система становится нестабильной
4. **Нестабильность процесса** — некорректные тесты вызывают отказы
5. **Снижение производительности всей системы** — цепочка ломается
6. **Негативное влияние на пользователя** — результаты неэффективны
7. **Увеличение времени исправления** — вместо 1x получаем 3-5x

### Риски от Gemini (расширенные)

#### 1. Groupthink (Групповое мышление)
- Все агенты на одной модели → одинаковые "слепые зоны"
- Generator выдает галлюцинацию → Critic подтверждает → 100% "зеленых" тестов, ничего не тестирующих
- **Решение:** Гетерогенные системы (разные модели для разных ролей)

#### 2. Fixer Loop Paradox (Зацикливание)
- Generator выдает плохой тест → Fixer добавляет еще больше галлюцинаций
- Бесконечный цикл в Docker, сжигающий токены
- **Решение:** Жесткий лимит итераций (max_retries) + RAG-валидация библиотек

#### 3. Test Suite Erosion (Эрозия тестов)
- Coverage растет, но Mutation Score падает
- Тесты становятся слишком простыми (только "happy path")
- **Решение:** Mutation Testing как "верховный судья"

#### 4. RAG Poisoning (Отравление памяти)
- Фиксер чинит "костылем" (time.sleep) → попадает в ChromaDB → предлагается как стандартное решение
- **Решение:** Ручная модерация + "карантин" для новых решений

#### 5. Objective Drift (Уход от цели)
- Агент оптимизирует не качество, а "успешность прохождения"
- Генератор пишет `assert True` → Критик пропускает
- **Решение:** Строгие промпты для Critic

### Сводная таблица

| Риск | Влияние | Контрмера |
|------|---------|----------|
| Groupthink | Критическое | Разные модели: Claude для Generator, GPT для Critic |
| Token Burn | Финансовое | parse_traceback + лимит итераций |
| RAG Poisoning | Долгосрочное | Чистка PATTERNS + Mutation Score верификация |
| Docker Escape | Безопасность | non-root + --network none |

### Ключевой индикатор

> **Mutation Score — единственный объективный индикатор правды.**
> Если низкий → система "сговорилась" или схалтурила.
