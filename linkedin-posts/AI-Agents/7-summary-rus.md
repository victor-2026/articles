# Статья 7: Ты измеряешь не то 🎯

**Контролируемый эксперимент: 3 навыка, 36 запусков, 2 проекта. Результат: 0% прироста.**

Мы проверили, повышают ли AI-скиллы (инструкции для модели) качество тестов. Три скилла × два режима (с/без) × три запуска × два проекта = 36 замеров.

**Проекты:** OrangeHRM (PHP/jQuery, legacy) и Buzzhive (React/Go API, modern). Оба дали 0% прироста.

**Скиллы:**
- **fault-injection** — мутационное тестирование (null через page.route). Результат: ни один null не пойман — оба фреймворка (PHP и React) обрабатывают null gracefully. Это не баг.
- **rest-api-qa** — лучшие практики API-тестов. Модель уже знает все паттерны — скилл добавил только гигиену (timeouts, URL verification), не корректность.
- **bdd-gherkin** — синтаксис Gherkin. Модель знает Given-When-Then наизусть — скилл даёт словарь и стиль, не правильность.

**Вывод:** Скиллы ≠ прирост корректности на известных паттернах. Они дают: консистентность, словарь, гигиену, онбординг. Но если модель уже знает паттерн, скилл — это гайд по стилю, а не новый функционал.

**Где скиллы реально нужны:** Там, где модель падает без них — контрактное тестирование, инфраструктура мутаций, сложные OAuth-потоки, финансовые state-машины.

**Главный урок:** Не спрашивай «помогает ли скилл?» — спроси «где модель ошибается без скилла?»

**Часть 2:** Архитектура Антона Гулина (3 слоя) — скиллы как валидаторы, а не генераторы.
