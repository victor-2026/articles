🚀 Как я улучшил качество QA, удалив 4,000 строк AI-тестов. Мой лучший результат за год.

💡 Суть: Иногда меньше AI-кода = больше качества

📊 Ключевые результаты рефакторинга:
• 277 уникальных тестов → 1,108 запусков в 4 браузерах
• API покрытие: 94% | PBT: 100%
• Время CI: 30 мин → 3 мин (12 эндпоинтов via GitHub Actions)
• Все на $0 AI бюджете

🔍 Проблема: Два монолитных файла — 3,908 строк AI-генерированного "спагетти"
• 15 дублированных блоков API
• Нет общих фикстур
• Сложно поддерживать и ревьють

✨ Решение: Один спринт на рефакторинг
• 23 чистых модуля вместо 2 монолитов
• Общий fixtures.ts с共享 настройками
• Стандартизированная структура и именование
• Тестовое покрытие не только сохранилось — стало надежнее

🐛 Неожиданный бонус: Нашли баг, месяц живший в продакшене
• Гонка условий в JWT-токенах обновления
• Последовательные тесты не ловили — одинаковые микросекунды → одинаковые exp
• Только параллельный запуск 4 браузеров показал коллизию
• Фикс: одна строка кода `jti: uuid.uuid4()`

💰 Инфраструктура:
• Docker (полный стек локально) + Render (бэкенд-only, бесплатно)
• Стабилизация бэкенда была обязательным пререквизит: exception handlers, connection pooling, uptime monitor
• GitHub Actions: smoke-тесты при каждом push

🔁 Экспериментировали с 15 AI-моделями — 6 уже устарели
Бесплатный AI — это беговая дорожка: нужно двигаться, чтобы не остаться позади.
