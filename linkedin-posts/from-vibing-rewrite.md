# From Vibing to Production: What I learned when I let AI write a production PR

**Rewrite — replaces original (May 28, 0.4% engagement, 8d old)**
**Format:** Post / LinkedIn Feed

---

I asked AI to write a production pull request. 87 lines of code in 14 seconds, compiled first try.

Then I looked at the tests: 3 assertions — all happy path. Zero edge cases. Zero error handling. Zero security checks.

I call this **Wipe Coding** — the gap from vibe (exploration) to wipe (production readiness). AI is fast at generation, blind at verification.

**Lessons:**
- AI generates code faster than I can review it — that's new
- Test generation is where AI falls apart most visibly
- Code review becomes the bottleneck, not coding
- "Works" ≠ "ready for production" — and AI can't tell the difference

**My takeaway:** The future is "Yes, And." Vibe coding for exploration. Wipe coding for production. The engineer becomes the reviewer, not the writer.
